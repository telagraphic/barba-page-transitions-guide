Containers

The container property on the data object holds the html to replace the data-container attribute in the html markup. See the docs for what I'm talking about. It is a best practice to remove the current.container when leaving a page to make way for the next.container on next pages' enter hook.

Toggle between the two links and you'll see how the transition doesn't work right, it's jolted. Sometimes you'll see overlapping pages that transition at the wrong time depending on your fades. Make sure to include current.container.remove() in one of your leave hooks.

Uncomment line 24 in app.js to see this working correctly. You'll notice that with this line missing, the inspector tab in your dev tools shows two data-barba="container" in the DOM. Compare this when current.container.remove() is uncommented in the leave hook.
