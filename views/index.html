Views

Views allow us to target a specific page. They are great for customizing animations or doing logic for a certain page. They are pretty straight-forward.

Checkout the code for some animations on the /guide page. I've set the selector .toc__chapter to opacity: 0 in the stylesheet and then used GSAP in a view namespace to apply a staggering fade with the afterEnter hook.

On the /resources page, I grabbed an attribute on the container and set the page header with it using a beforeEnter hook.

Hopefully this gives you a good idea of what these can be used for...
